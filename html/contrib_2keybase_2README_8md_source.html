1 # KeybaseMutt
2 
3 A work in progress. KeybaseMutt exists to let its users utilize Keybase within mutt. (Unfortunately, not natively (yet).)
4 
5 ## Quick start
6 
7 Run the install script. It will set up the directory's and paths for you. Do what it says or suffer the consequences!
8 
9 Then just throw these into your .muttrc
10 
11 `set editor = 'echo %s > ~/.mutt/keybaseMutt/.tmp; vim %s'`
12 
13 `macro compose K "<enter-command>unset wait_key<enter><shell-escape>python ~/.mutt/keybaseMutt/keybase.py<enter><enter-command>set wait_key<enter>`
14 
15 Done!

17 ## How to use KeybaseMutt
18 
19 Write an email. In the screen right before you send your email (called the pager mode), press "K".
20 
21 ![](pagerMode.png)
22 (Press 'K' when you're here)
23 
24 You can now use four Keybase commands in this "shell". (I thought it best to limit the user to four commands for security reasons. Running unusual commands would overwrite the email, forcing the user to rewrite the email again.)
25 
26 The commands are:
27 - `keybase encrypt [user]`
28 - `keybase pgp encrypt [user]`
29 - `keybase sign`
30 - `keybase pgp sign`

33 ## Decrypting and verifying messages
34 
35 Unfortunately, there isn't an easy way to decrypt or verify messages through a macro. Instead, you'll need to use the pipe feature of mutt.
36 
37 Opening the email, you'll need to navigate to the actual attachment. (Press "V" when you're reading the email.)
38 
39 ![](attachmentView.png)
40 
41 Then pipe ("|") it through the one of the scripts. (like "pgpverify.sh" or "decrypt.sh")
42 
43 Please note, using an attachment is very important when decrypting emails that include html. The scripts are unable to seperate html and text.
44 
45 Make sure that the attachment is "text/plain" rather than "text/html".
